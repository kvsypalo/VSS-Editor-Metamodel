= Manage Schema Workspace State

== Краткое описание
В этом дочернем домене Актор удерживает схему как рабочий файл и рабочую сессию: начинает новую схему, возвращается к существующей, подмешивает внешний фрагмент, выгружает результат наружу и сохраняет текущую картину, если входной JSON оказался ошибочным.

== Текущее состояние
Сценарий уже реализован как группа соседних команд вокруг одного и того же workspace state. Все операции идут через schema menu и не требуют перехода в отдельные экраны или внешние сервисы.

== Что подтверждено
- `New` и `Open` формируют базовое рабочее состояние схемы.
- `Add Schema`, `Export JSON` и `Export PNG` работают как продолжение той же сессии.
- Ошибка чтения JSON завершает входную операцию отказом и сохраняет текущую схему на месте.

== Замечания аналитика
Детальные user routes уже materialized в родительском домене. Здесь они собраны в один child-domain scenario, потому что для decomposition важен именно единый lifecycle рабочей схемы как предметная граница.

== Требуемые изменения
Новых изменений не требуется.
