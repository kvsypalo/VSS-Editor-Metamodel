= Derive Flow Type And Routing From Endpoints

== Краткое описание
Тип flow и его визуальный маршрут выводятся из того, какие endpoints заданы у связи. Наличие source и target node делает flow `internal`, отсутствие source при наличии target делает `external_in`, отсутствие target при наличии source делает `external_out`, а правонаправленные назад и self-loop связи рисуются как ломаные вместо стандартной кривой.

== Текущее состояние
Правило уже реализовано в `autoTypeFlow()` и `drawFlow()`. Редактор автоматически пересчитывает тип при изменении source/target nodes и выбирает геометрию линии исходя из положения узлов и self-loop случая.

== Что подтверждено
- `autoTypeFlow()` пересчитывает `flow.type` по комбинации source/target node IDs.
- `drawFlow()` рисует right-to-left и self-loop flows как ломаные с вертикальным обходом.
- Остальные flows строятся как плавная cubic curve с arrowhead и optional label.

== Замечания аналитика
Правило одновременно покрывает semantic классификацию flow и observable routing behavior, потому что оба аспекта вычисляются из одной и той же endpoint-конфигурации.

== Требуемые изменения
Новых изменений не требуется.
