= Import Prefixes Colliding Identifiers

== Краткое описание
При import дополнительной схемы editor не допускает collisions между существующими и imported identifiers. Если ID уже существует в текущем state, к imported object добавляется случайный префикс `impXXXX_`, а связанные references remap-ятся на новые значения.

== Текущее состояние
Правило реализовано в `importWithPrefix(imported)`. Оно отдельно отслеживает существующие node, flow, element и text IDs и применяет remap до добавления imported objects в текущую схему.

== Что подтверждено
- Префикс генерируется как `imp` + случайный four-char suffix + `_`.
- Node и flow ID remap-ятся через `nodeIdMap` и `flowIdMap`.
- Element и text objects получают remapped IDs, а `node_id` и `flow_id` у imported elements переписываются на новые значения.

== Замечания аналитика
Правило фиксирует structural merge safety, но не вводит semantic deduplication imported domain objects.

== Требуемые изменения
Новых изменений не требуется.
