= Export Schema Artifacts

== Краткое описание
Актер завершает очередной цикл работы и выводит схему наружу в том формате, который нужен дальше по работе. JSON нужен, чтобы вернуться к той же схеме позже и продолжить редактирование, а PNG - чтобы показать текущую картину коллегам, приложить к обсуждению или использовать как наглядный снимок состояния.

== Текущее состояние
Сценарий уже реализован как две соседние команды одного и того же рабочего шага. Актер не покидает редактор, а просто выбирает, нужен ли ему рабочий файл для продолжения редактирования или визуальный результат для показа.

== Что подтверждено
- Export JSON использует имя текущей схемы и сохраняет весь schema state.
- Export PNG выводит саму диаграмму как отдельный визуальный результат.
- Если диаграмма пуста, редактор явно сообщает, что экспортировать нечего.

== Замечания аналитика
Две export-команды объединены в один сценарий, потому что для пользователя это один и тот же шаг завершения работы: вынести текущую схему из редактора в нужном формате.

== Требуемые изменения
Новых изменений не требуется.
