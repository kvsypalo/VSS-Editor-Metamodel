= Canvas Authoring Domain Map

== Краткое описание
Дочерний домен `canvas-authoring` описывает непрерывное уточнение схемы на рабочем холсте: редактирование converters, flows, element types и text annotations, а также прямую работу с properties panel, drag, selection, zoom и pan.

Этот child domain наследует value contract родительского домена `vss-editor` и не materialize собственный layer `01`, потому что Актор и прикладная ценность здесь совпадают с общим authoring route продукта.

== Текущее состояние
Домен выделен из общего product scope, чтобы удерживать отдельную ответственность за структуру графа, редактирование свойств и интерактивное поведение canvas. Source basis остается той же - web bundle `index.html`, `app.js`, `styles.css` из каталога `C:\MetaModel\Example code\vss-editor-2.4`.

== Что подтверждено
- Canvas и properties panel образуют единый authoring surface без отдельных экранов.
- Актор может добавлять и менять converters, flows, value elements и text annotations.
- Правки на canvas опираются на локальные graph rules родительского домена и сразу видимы в рабочей схеме.

== Замечания аналитика
Child domain выделен по смысловой границе непрерывного authoring cycle. Он отличается от file/workspace lifecycle собственными interaction rules и собственной change surface.

== Требуемые изменения
Новых изменений продукта не запрошено. Требуется явно materialize child domain по канону repository structure и publication view.
