= Workspace Lifecycle Domain Map

== Краткое описание
Дочерний домен `workspace-lifecycle` описывает, как Актор получает редактируемую рабочую схему, дополняет ее внешним JSON-фрагментом, выносит результат в файл и сохраняет сессию устойчивой при ошибке чтения входного JSON.

Этот child domain наследует value contract родительского домена `vss-editor` и не materialize собственный layer `01`, потому что здесь нет отдельного Актора или самостоятельного value movement вне общего авторского маршрута редактора.

== Текущее состояние
Домен выделен из общего product scope, чтобы удерживать самостоятельную ответственность за вход в рабочую сессию, file-based операции и failure boundary по невалидному JSON. Source basis для домена остается той же - web bundle `index.html`, `app.js`, `styles.css` из каталога `C:\MetaModel\Example code\vss-editor-2.4`.

== Что подтверждено
- `New` и `Open` переводят Актора в пригодное к редактированию состояние схемы.
- `Add Schema`, `Export JSON` и `Export PNG` образуют локальный контур обмена схемой с внешним файловым миром.
- Parse error на входном JSON завершает операцию отказом и сохраняет текущую рабочую схему.

== Замечания аналитика
Дочерний домен выделен по смысловой границе рабочего жизненного цикла схемы. Он меняется отдельно от canvas interaction rules и может развиваться автономно как контур file/session behavior.

== Требуемые изменения
Новых изменений продукта не запрошено. Требуется явно materialize child domain по канону repository structure и publication view.
