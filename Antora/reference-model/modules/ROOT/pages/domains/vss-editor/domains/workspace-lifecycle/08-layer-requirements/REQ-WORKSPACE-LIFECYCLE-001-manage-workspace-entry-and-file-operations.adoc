= Manage Workspace Entry And File Operations

== Краткое описание
Редактор должен удерживать рабочую схему как единое editable workspace state. Актор должен иметь возможность начать новую схему, открыть существующую, добавить внешний фрагмент, выгрузить результат и безопасно пережить ошибочный входной JSON без потери уже открытой работы.

== Текущее состояние
В текущей реализации все перечисленные операции уже живут в schema menu и вокруг общего `state.schema`. `New`, `Open`, `Add Schema`, `Export JSON`, `Export PNG` и invalid-JSON handling образуют один локальный contract child domain.

== Что подтверждено
- Entry routes и file routes используют общий browser-only workspace state.
- Import и export меняют или выносят схему без переключения в другой runtime context.
- Parse error не разрушает текущее рабочее состояние.

== Замечания аналитика
Requirement child domain удерживает именно локальный lifecycle contract. Детализированные acceptance branches остаются раскрытыми в родительских `REQ-VSS-EDITOR-*` и `SCN-VSS-EDITOR-*`.

== Требуемые изменения
Новых изменений не требуется.
