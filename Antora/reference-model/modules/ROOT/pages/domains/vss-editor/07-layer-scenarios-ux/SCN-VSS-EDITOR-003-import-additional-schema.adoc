= Import Additional Schema

== Краткое описание
Актор берет еще один фрагмент схемы и добавляет его в уже открытую работу через `Add Schema`. Сценарий нужен не для замены текущей картины, а для ее расширения: собрать в одном canvas несколько частей системы и дальше править их вместе.

== Текущее состояние
Сценарий уже реализован как добавление второй схемы к текущей рабочей картине. Если у объектов совпадают идентификаторы, редактор разводит их автоматически, чтобы Актор получил единый рабочий canvas с корректными ссылками.

== Что подтверждено
- `btnImport` использует тот же file input, что и `Open`, и запускает сценарий объединения схемы с текущей рабочей картиной.
- `importWithPrefix()` добавляет в текущую работу nodes, flows, elements и texts из второго файла.
- При дублировании ID редактор автоматически разводит imported objects и обновляет их связи.

== Замечания аналитика
Source basis не подтверждает более сложные merge rules вроде semantic deduplication или объединения одноименных объектов по смыслу.

== Требуемые изменения
Новых изменений не требуется.
