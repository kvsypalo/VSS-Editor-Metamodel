= Reject Invalid JSON Without State Loss

== Краткое описание
Если Актор пытается открыть или импортировать невалидный JSON, редактор завершает операцию явным сигналом ошибки и сохраняет текущую схему доступной для продолжения работы.

== Текущее состояние
`fileInput.onchange` оборачивает `JSON.parse` в `try/catch`. При ошибке парсинга редактор показывает `alert("Invalid JSON")`, сохраняет текущий state и завершает операцию без загрузки файла.

== Что подтверждено
- Открытие и import используют общий обработчик `fileInput.onchange`.
- При exception вызывается только `alert("Invalid JSON")`.
- Мутация `state.schema` выполняется только в ветках успешного parse.

== Замечания аналитика
Source basis подтверждает защиту от parse error и сохранение текущей сессии. Дополнительная recovery UX и field-level validation схемы в текущем scope не материализованы.

== Требуемые изменения
Новых изменений не требуется.
