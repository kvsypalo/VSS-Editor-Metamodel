= Create And Open Schema State

== Краткое описание
Редактор должен позволять начать работу либо с новой пустой схемы по умолчанию, либо с существующей JSON-схемы, загруженной из файла. Обе точки входа должны переводить пользователя в редактируемое состояние canvas.

== Текущее состояние
Команды `New` и `Open` доступны из `Schema` menu. `New` пересоздает schema state через `createEmptySchema()`, а `Open` читает JSON-файл через `FileReader`, нормализует его и заменяет текущую схему.

== Что подтверждено
- `New` создает schema с `schema`, `nodes`, `flows`, `elements` и `texts`.
- `Open` использует hidden `fileInput`, парсит JSON и при успехе выполняет `normalizeSchema`, `selectSchema`, `centerView` и `render`.
- После обоих действий пользователь оказывается в состоянии, пригодном для дальнейшего редактирования.

== Замечания аналитика
Source basis не показывает валидацию пользовательского JSON beyond successful parsing and normalization. Requirement фиксирует только подтвержденное поведение текущей реализации.

== Требуемые изменения
Новых изменений не требуется.
