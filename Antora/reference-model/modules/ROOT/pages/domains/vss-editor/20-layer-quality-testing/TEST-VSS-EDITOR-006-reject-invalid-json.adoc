= Reject Invalid JSON Test

== Краткое описание
Тест проверяет, что open/import невалидного JSON не повреждает текущую схему и дает явный сигнал ошибки. Проверка закрывает failure scenario с parse error.

== Текущее состояние
Тест materialized как validating specification по общему `try/catch` вокруг `JSON.parse` в обработчике `fileInput.onchange`.

== Что подтверждено
- Parse error приводит к `alert("Invalid JSON")`.
- Ни ветка `open`, ни ветка `import` не выполняют state mutation после exception.
- Failure route доступен для обоих пользовательских entry points: `Open` и `Add Schema`.

== Замечания аналитика
Тест фиксирует отказную ветку текущей реализации без предположений о richer recovery UX.

== Требуемые изменения
Новых изменений не требуется.
