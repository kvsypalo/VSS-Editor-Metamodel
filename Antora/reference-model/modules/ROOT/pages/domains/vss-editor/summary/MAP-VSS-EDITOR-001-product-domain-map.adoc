= VSS Editor Product Domain Map

== Краткое описание
Домен `vss-editor` описывает статический браузерный редактор схем потока ценности Æilus. Продукт позволяет собрать и отредактировать схему из converters, flows, element types и текстовых пометок, а затем выгрузить результат в JSON или PNG.

Первая wave materialize подтвержденный code-driven scope верхнеуровневого домена и теперь разложена на два child domains. `workspace-lifecycle` удерживает вход в рабочую сессию, файловые операции и отказной маршрут по невалидному JSON, а `canvas-authoring` удерживает непрерывное редактирование структуры схемы на canvas.

== Текущее состояние
В репозитории эталонной модели до этой wave не было product-local домена для VSS Editor. Исходная source basis ограничена web bundle `index.html`, `app.js`, `styles.css` и встроенной help-модалью продукта из каталога `C:\MetaModel\Example code\vss-editor-2.4`.

== Что подтверждено
- Продукт работает как browser-only single-page editor без серверной части и без build-step.
- Родительский домен удерживает общий value/process/application/access/critical-path контур, а предметные поведенческие ветки дополнительно materialized в child domains `workspace-lifecycle` и `canvas-authoring`.
- В домене отдельно materialized слой приложения `10`, baseline-оценка доступа `31` и оценка critical path `33`.

== Замечания аналитика
Source basis не включает runtime telemetry, backend contracts, user interviews и formal product documentation. Поэтому в этой wave materialized только те слои, которые прямо подтверждаются UI и кодом продукта.

== Требуемые изменения
Новых product changes не запрошено. Требуется formalize текущую подтвержденную модель продукта в approval-ready виде.
