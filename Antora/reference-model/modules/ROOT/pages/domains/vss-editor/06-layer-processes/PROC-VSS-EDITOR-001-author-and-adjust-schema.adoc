= Author And Adjust Schema

== Краткое описание
Это верхнеуровневый процесс работы со схемой ценности. Он начинается с того, что Актор входит в редактор с пустым холстом или уже подготовленным JSON-файлом, затем собирает или уточняет схему на canvas, при необходимости подмешивает второй фрагмент через import и завершает цикл выгрузкой результата в JSON или PNG.

У процесса есть понятный старт, рабочая середина и финиш. Стартом является появление редактируемой схемы в редакторе, рабочая часть состоит из последовательных правок converters, flows, element types и заметок, а финишем становится переносимый результат, который можно открыть снова или вынести в обсуждение.

== Текущее состояние
Весь процесс выполняется внутри одной браузерной страницы и держится на текущем schema state в памяти. Отдельного серверного хранения или фоновой синхронизации нет: если Актору нужен результат за пределами текущей сессии, процесс должен завершиться явным экспортом.

== Что подтверждено
- Процесс начинается через `Schema` menu: Актор создает новую схему, открывает существующую или добавляет еще один JSON-фрагмент в текущую работу.
- Основная работа идет на canvas и в properties panel: Актор меняет состав схемы, параметры объектов и связи между ними.
- Процесс завершается export-командой, которая превращает текущую схему в JSON или PNG.

== Замечания аналитика
Исходники достаточно малы, поэтому процесс пока остается одной верхнеуровневой карточкой. В таком виде он фиксирует старт, участников, шаги, результат и отказной путь.

== Требуемые изменения
Новых изменений не требуется.
