= 15 Technology Architecture

== Path
`domains/<domain>/15-layer-technology-architecture/`

== Purpose
Слой фиксирует технологический стек, платформы, инфраструктурные технологии и ключевые technical choices.

== Activation
- Этот слой входит в канонический стек `01-33`.
- Домен может пометить слой как `unused`, если локально не фиксирует отдельные technology cards.

== Typical objects
- `TECH-*`

== Special fields
- Отдельные layer-specific поля не заданы. Используется `BASE-CARD-SCHEMA-001`.

== Traceability rules
- В root-правилах нет отдельного минимального link contract специально для `TECH-*`.
- Технологические решения должны сохранять явные связи с `APP-*`, `OPS-*`, `NFR-*` и `DEBT-*`, если они задают platform constraints.

== Key rules
- Технологические решения оформляются как устойчивые platform choices и ограничения.
- Они должны быть связаны с приложениями, эксплуатацией, NFR и техническим долгом.

== Related layers
- Upstream: `10 Applications Architecture`, `08 Requirements`, `17 NFR Quality Attributes`.
- Downstream: `16 Operations`, `19 Technical Debt`, `32 Performance`.

== Authoring notes
- Root-правила не задают дополнительных authoring notes для этого слоя кроме `CARD-CREATION-RULES-001` и `BASE-CARD-SCHEMA-001`.
