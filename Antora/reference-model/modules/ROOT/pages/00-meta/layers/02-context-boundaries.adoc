= 02 Context Boundaries

== Path
`domains/<domain>/02-layer-context-boundaries/`

== Purpose
Слой фиксирует контекст продукта или домена, его границы ответственности, внешние и внутренние boundaries, а также важные точки стыка с соседними системами и доменами.

== Activation
- Этот слой входит в канонический стек `01-33`.
- Домен может пометить слой как `unused`, если локально не фиксирует отдельные контекстные карточки и границы уже выражены в других канонических объектах.

== Typical objects
- `CTX-*`

== Special fields
- Отдельные layer-specific поля не заданы. Используется `BASE-CARD-SCHEMA-001`.

== Traceability rules
- В root-правилах не задано дополнительных минимальных traceability links специально для этого слоя.
- Связи оформляются через базовые `relations` и через явные ссылки на требования, интеграции и доменную декомпозицию, если они опираются на контекстные объекты.

== Key rules
- Граница ответственности и scope домена должны быть выражены явно.
- Контекстные объекты служат опорой для требований, интеграций и доменной декомпозиции.

== Related layers
- Downstream: `05 Business Architecture`, `08 Requirements`, `12 Integrations`.
- Supporting structure: доменная декомпозиция и локальные `00-meta` карты.
