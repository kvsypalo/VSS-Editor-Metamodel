= 00 Meta

== Path
`domains/<domain>/.summary/`

== Purpose
Локальный meta/context слой домена. Здесь фиксируются карта домена, роль домена, связь с родителем, ответственный за домен, используемые и неиспользуемые канонические слои, а также ссылка на принятую версию root `00-meta/`.

== Activation
- Этот слой обязателен для каждого верхнеуровневого и дочернего домена.
- Домен не может удалить, переименовать или заменить этот слой локальной альтернативой.

== Typical objects
- `MAP-*`

== Special fields
- `domain_role`: например `reference`, `product`, `child-domain`
- `parent_domain`: обязателен во всех дочерних доменах
- `reference_release`
- `used_layers`: список фактически используемых канонических layer codes
- `unused_layers`: список канонических layer codes, которые домен явно не использует

== Traceability rules
- Каждый дочерний домен должен иметь явную связь с родительским доменом и собственные объекты минимум в `.summary`, `08`, `24` и `25`.
- Локальный домен не может менять канонический набор слоев и обязательные root-правила.

== Key rules
- Локальный `.summary/` наследует root `00-meta/` и не меняет канонический набор слоев.
- Домен может явно отметить слой как `unused`, но не может удалить, переименовать или локально переопределить канонический слой.
- Каждый домен должен иметь явного ответственного, зафиксированного в `MAP-*` через поле `owner`.
- Любая domain wave стартует в своем домене и только после impact-check каскадируется в root, parent или sibling domains.
- Root `00-meta/` остается единственным источником repository-wide governance, schemas, naming rules, templates и канона слоев.

== Related layers
- Downstream: все канонические слои `01-33`.
- Supporting root areas: `repository-structure/`, `schemas/`, `templates/`, `traceability-rules/`, `naming-rules/`, `governance/`.

== Authoring notes
- В дочернем домене `parent_domain` должен совпадать с фактическим положением домена в дереве репозитория.
- `used_layers` и `unused_layers` должны быть заполнены явно; молчаливое отсутствие слоя не заменяет декларацию.
- В `owner` фиксируй реальную ответственную роль или владельца домена, например `product`, `analysis`, `architecture` или конкретную команду.
- Если wave начинается в этом домене и затрагивает соседей или родителя, зафиксируй это в `MAP-*` или связанных `TRACE-*`/`CHANGE-*`, а не оставляй cascade неявным.
