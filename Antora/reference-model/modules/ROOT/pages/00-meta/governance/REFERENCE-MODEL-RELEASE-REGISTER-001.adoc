= REFERENCE-MODEL-RELEASE-REGISTER-001

Реестр release identity эталонной модели.

== 1. Назначение
- зафиксировать канонический формат release identifier эталона
- хранить список опубликованных release scope эталонной модели
- дать project repos машиночитаемую и человекочитаемую точку ссылки на принятую версию эталона

== 2. Формат release identifier
Канонический формат:

`RM-YYYYMMDD-NNN`

Где:
- `RM` - reference model
- `YYYYMMDD` - дата release scope
- `NNN` - порядковый номер release scope в пределах даты, начиная с `001`

== 3. Правила использования
- любое принятое изменение в root `00-meta/` создает новый release scope
- каждый release scope должен получить новый `RM-YYYYMMDD-NNN`
- идентификатор фиксируется в этом реестре и должен совпадать с release tag или release identifier, используемым в Git
- project repo обязан ссылаться именно на значение из этого реестра
