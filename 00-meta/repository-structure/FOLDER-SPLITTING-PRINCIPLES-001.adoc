= FOLDER-SPLITTING-PRINCIPLES-001

== Принципы разбиения
- сначала репозиторий делится на домены, а не только на процессы
- новый домен создается по смысловой границе, а не только по размеру кода или числа файлов
- дочерний домен создается, если у него есть собственная ответственность, локальная терминология или отдельный темп изменений
- grouped view канонического набора слоев хранится в `../layers-summary/LAYER-GROUPS-001.adoc`

== Наследование структуры
- корневой `00-meta/` задает эталонную структуру
- верхнеуровневый домен наследует эталонную структуру всегда
- дочерний домен наследует структуру родительского домена всегда
- дочерний домен может не использовать часть слоев, но не может вводить свои слои локально
- при изменении канонического набора слоев обновляются и `00-meta/layers/`, и `00-meta/layers-summary/`

== `REF-*`
Создавай локальный reference-object, если объект уже существует выше и нужен для локальной навигации.
Не создавай локальный reference-object, если объект локальный или хватает прямой связи в карточке.
